# Google Search Console & SEO 가이드라인 종합 문서

> **목적**: 이 문서는 Google 공식 가이드라인을 기반으로 정리한 SEO/검색 최적화 참고 문서입니다.
> 새로운 기능 추가, 콘텐츠 수정, 페이지 삭제 등 모든 작업 시 이 문서를 참고하여 Google 정책을 준수해야 합니다.
> **출처**: https://developers.google.com/search/docs (2026년 4월 기준)

---

## 1. 유용한 콘텐츠 제작 가이드라인

### 1-1. 핵심 원칙
Google의 자동 순위 시스템은 **"사람들에게 도움을 주기 위해 제작된 유용하고 신뢰할 수 있는 정보"**를 우선시한다.
검색엔진 순위 조작 목적의 콘텐츠는 보상 대상이 아니다.

### 1-2. 콘텐츠 품질 체크리스트
- [ ] 고유한 정보, 조사, 분석 자료를 제공하는가?
- [ ] 주제에 대한 본질적이고 완전한 설명을 제공하는가?
- [ ] 뻔하지 않은 유용한 분석이나 흥미로운 정보를 포함하는가?
- [ ] 다른 출처를 활용할 때 단순 복사가 아닌 가치와 독창성을 추가했는가?
- [ ] 제목이 이해를 돕는 요약인가? (과장/충격적이지 않은가?)
- [ ] 북마크, 공유, 추천할 만한 가치가 있는 페이지인가?
- [ ] 철자/스타일 오류가 없는가?
- [ ] 완성도 높게 제작되었는가? (급하게 제작되지 않았는가?)

### 1-3. E-E-A-T 원칙
**경험(Experience) + 전문성(Expertise) + 권위(Authority) + 신뢰성(Trustworthiness)**

- **신뢰성이 가장 중요**하며, 나머지는 신뢰성에 기여하는 요소
- 작성자 정보(바이라인)를 명확히 표시
- 작성자의 배경과 전문 분야를 설명
- AI/자동화 사용 시 투명하게 공개

### 1-4. 검색엔진 중심 콘텐츠 경고 신호 (피해야 할 것)
- 주로 검색엔진 방문 유도 목적인 콘텐츠
- 여러 주제에 대해 대량 콘텐츠 제작
- 광범위한 자동화로 다양한 주제 콘텐츠 생성
- 다른 사람의 이야기를 요약하며 가치를 더하지 않는 것
- 실제 변경 없이 페이지 날짜만 변경하는 것
- 특정 단어 수에 맞추려는 것 (Google은 선호 단어 수가 없음)

## 2. SEO 기본 가이드

### 2-1. Google이 콘텐츠를 찾도록 하기
- `site:도메인` 검색으로 색인 상태 확인
- 사이트맵 제출 (필수는 아니지만 권장)
- CSS와 JavaScript 등 중요 리소스 접근 가능하게 유지
- Search Console의 URL 검사 도구로 확인

### 2-2. 사이트 구성
- **설명적 URL 사용**: `example.com/pets/cats.html` (임의 식별자 피하기)
- **주제별 페이지 그룹화**: 디렉터리로 비슷한 주제 분류
- **중복 콘텐츠 축소**: `rel="canonical"` 또는 301 리디렉션으로 표준 URL 지정

### 2-3. 콘텐츠 품질 기준
- **가독성**: 긴 콘텐츠는 단락/섹션으로 나누고 제목 제공
- **독창성**: 다른 사이트의 콘텐츠를 복사하지 않기
- **최신성**: 이전 콘텐츠를 검토하고 필요에 따라 업데이트/삭제
- **신뢰성**: 전문성이나 경험의 출처 제시

### 2-4. 검색결과 표시 최적화
- **제목(title)**: 명확하고 간결하며 페이지 콘텐츠를 정확히 설명
- **메타 설명**: 짧고 명확하며 각 페이지에 고유하게 작성
- **이미지**: 관련 텍스트 근처에 배치, alt 속성에 설명적 텍스트 추가

### 2-5. 무시해야 할 것들
- **메타 키워드 태그**: Google은 사용하지 않음
- **키워드 반복**: 과도한 반복은 스팸 정책 위반
- **도메인 이름의 키워드**: 순위에 거의 영향 없음
- **콘텐츠 길이**: 최소/최대 단어 수 요구사항 없음
- **제목 태그 순서**: Google 검색 관점에서 순서는 무관

## 3. Google 검색 필수사항 (Search Essentials)

### 3-1. 기술 요구사항
- Google이 웹페이지에 접근 가능해야 함
- robots.txt와 메타 태그로 크롤링 제어
- 모바일 최적화 필수
- 적절한 파일 형식 사용 (HTML, PDF, 이미지, 동영상 등)

### 3-2. 주요 권장사항
- 유용하고 신뢰할 수 있는 사용자 중심 콘텐츠 제작
- 사용자 검색어를 고려한 단어 사용
- 링크를 크롤링 가능하도록 설정
- 대체 텍스트와 링크 텍스트 작성
- 사이트맵 제출
- 구조화된 데이터 활용
- noindex 메타 태그로 검색 제외 콘텐츠 관리

## 4. 스팸 정책 (절대 위반하면 안 되는 것들)

### 4-1. 클로킹
검색엔진과 사용자에게 서로 다른 콘텐츠를 표시하는 것은 금지.

### 4-2. 키워드 스터핑
순위 조작 목적으로 웹페이지를 키워드로 채우는 것은 금지.

### 4-3. 숨겨진 텍스트/링크
- 흰색 배경에 흰색 텍스트
- CSS로 화면 밖 배치
- 글꼴 크기/불투명도 0 설정
- **허용**: 탭/아코디언으로 추가 콘텐츠 숨기기, 스크린 리더 전용 텍스트

### 4-4. 링크 스팸
- 순위 조작 목적 링크 구매/판매 금지
- 과도한 링크 교환 금지
- 광고/스폰서십 링크에는 `rel="nofollow"` 또는 `rel="sponsored"` 사용

### 4-5. 대량 콘텐츠 악용 (Scaled Content Abuse)
- 생성형 AI로 가치 없이 대량 페이지 생성 금지
- 스크래핑으로 가치 없는 다수 페이지 생성 금지
- 자동 변환(동의어, 번역)으로 대량 페이지 생성 금지

### 4-6. 스크래핑
다른 사이트 콘텐츠를 복사하여 독자적 가치 없이 게시하는 것은 금지.

### 4-7. 부적절한 리디렉션
사용자를 원래 요구와 다른 콘텐츠로 악의적 리디렉션하는 것은 금지.

### 4-8. 사이트 인지도 악용
기존 순위 신호 활용 목적으로 관련 없는 서드파티 콘텐츠를 호스팅하는 것은 금지.

## 5. 페이지 경험 (Page Experience)

### 5-1. 자체 평가 체크리스트
- [ ] Core Web Vitals가 우수한가?
- [ ] HTTPS로 안전하게 제공되는가?
- [ ] 모바일 친화적인가?
- [ ] 과도한 광고가 없는가?
- [ ] 방해가 되는 전면 광고(인터스티셜)가 없는가?
- [ ] 주요 콘텐츠와 광고를 쉽게 구분할 수 있는가?

### 5-2. Core Web Vitals
- Search Console의 Core Web Vitals 보고서로 확인
- PageSpeed Insights로 페이지별 성능 측정
- Chrome Lighthouse로 종합 성능 평가
- **점수만으로 순위가 보장되지 않음** — 사용자 경험 향상이 진정한 목표

### 5-3. 광고 관련 주의사항
- 콘텐츠 접근을 가리는 팝업 금지
- 콘텐츠를 읽기 위해 광고를 닫아야 하는 방식 금지
- 스크롤 중 갑자기 나타나는 광고 금지
- **핵심 콘텐츠를 방해하거나 주의를 분산시키는 과도한 수의 광고 금지**

## 6. 구조화된 데이터 (Structured Data)

### 6-1. 형식
- **JSON-LD (권장)**: `<script>` 태그 내 삽입, 구현/유지보수 가장 용이
- 마이크로데이터, RDFa도 지원되나 JSON-LD 권장

### 6-2. 주요 스키마 유형
| 유형 | 용도 |
|------|------|
| Article | 기사, 블로그 게시물 |
| FAQ | 자주 묻는 질문 |
| HowTo | 단계별 가이드 |
| WebApplication | 웹 애플리케이션 |
| Organization | 조직 정보 |
| BreadcrumbList | 탐색경로 |
| Review | 리뷰 |
| Product | 제품 |
| Event | 이벤트 |

### 6-3. 가이드라인
- 필수 속성을 모두 포함해야 리치 결과로 표시
- 권장 속성을 많이 정의할수록 표시 가능성 증가
- **적지만 완전하고 정확한 데이터 > 많지만 부정확한 데이터**
- 페이지 콘텐츠를 설명하는 마크업만 사용 (사용자에게 보이지 않는 정보 마크업 금지)
- 빈 페이지에 구조화된 데이터만 보관하는 것 금지
- 리치 결과 테스트 도구로 유효성 검증

## 7. 사이트맵

### 7-1. 필요한 경우
- 500개 이상 페이지의 대규모 사이트
- 외부 링크가 적은 새로운 사이트
- 동영상/이미지/뉴스 등 리치 미디어 콘텐츠가 많은 경우

### 7-2. 형식
- XML 형식 표준 사이트맵
- 동영상/이미지/뉴스 확장 사이트맵
- CMS 사용 시 자동 생성 가능

### 7-3. 주의사항
- 사이트맵이 있어도 모든 항목이 크롤링/색인 생성된다고 보장하지 않음
- 하지만 대부분의 경우 도움이 됨

## 8. 중복 URL 통합 (Canonical)

### 8-1. 방법 (효과 순서)
1. **301/308 리디렉션** (가장 강력)
2. **`rel="canonical"` 링크 태그** (HTML `<head>` 섹션)
3. **`rel="canonical"` HTTP 헤더** (PDF 등 비HTML)
4. **사이트맵** (약한 신호)

### 8-2. 모범 사례
- 절대 경로 사용 필수 (상대 경로 금지)
- `<head>` 섹션에만 포함
- robots.txt로 표준화 금지
- HTTPS를 HTTP보다 우선시
- 사이트 내 링크는 표준 URL로 통일

## 9. 다국어 사이트 (hreflang)

### 9-1. 구현 방법
```html
<link rel="alternate" hreflang="ko" href="https://example.com/ko/page" />
<link rel="alternate" hreflang="en" href="https://example.com/en/page" />
<link rel="alternate" hreflang="x-default" href="https://example.com/ko/page" />
```

### 9-2. 필수 규칙
- 각 언어 버전은 자신을 포함해 모든 다른 언어 버전을 나열해야 함
- **양방향 링크 필수**: A→B이면 B→A도 있어야 함
- 정규화된 전체 URL 사용 필수
- ISO 639-1 언어코드 + ISO 3166-1 지역코드 형식

### 9-3. 흔한 실수
- 반환 링크 누락 (양방향 참조 필수)
- 잘못된 언어/지역 코드 사용
- `x-default` 값 누락

## 10. Search Console 활용

### 10-1. 필수 설정
1. 사이트 소유권 확인

| 직접 조치 | 검색결과 제외 문제 확인 |
