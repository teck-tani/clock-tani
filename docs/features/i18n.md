# 국제화 (i18n)

## 목적

한국어(ko)와 영어(en) 2개 언어를 지원하는 다국어 시스템. next-intl 기반으로 URL에 locale 접두사를 포함하며, 서버/클라이언트 양쪽에서 번역을 사용한다. 한국 시장 우선, 해외 확장을 위한 구조.

## 파일 맵

| 구분 | 파일 경로 |
|------|-----------|
| 한국어 번역 | `messages/ko.json` |
| 영어 번역 | `messages/en.json` |
| i18n 설정 | `src/i18n/request.ts` |
| 네비게이션 설정 | `src/navigation.ts` |
| 미들웨어 | `src/middleware.ts` |
| 레이아웃 | `src/app/[locale]/layout.tsx` (NextIntlClientProvider) |

## 비즈니스 규칙

- 지원 locale: `ko` (기본값), `en`
- URL 형식: `/{locale}/{page}` (예: `/ko/timer`, `/en/alarm`)
- locale 접두사 항상 표시 (숨김 없음)
- `/` → `/ko` 자동 리다이렉트 (301)
- 미들웨어: `/((?!api|_next|_vercel|.*\\..*).*)` 패턴으로 라우팅
- 서버 컴포넌트: `getTranslations({ locale, namespace })` 사용
- 클라이언트 컴포넌트: `useTranslations(namespace)` 사용

## 번역 키 구조

```json
{
  "Common": { "Share": "공유", "Copied": "복사됨" },
  "Header": { "Menu": "메뉴", "Settings": "설정" },
  "Footer": { ... },
  "Meta": { "defaultTitle": "...", "defaultDescription": "..." },
  "Clock": {
    "Clock": { "meta": { "title": "...", "description": "..." }, "labels": { ... } },
    "Timer": { "meta": { ... }, "mode": { ... }, "controls": { ... } },
    "Alarm": { ... },
    "Stopwatch": { ... }
  },
  "Guides": {
    "titles": { ... },
    "descriptions": { ... },
    "articles": { "{slug}": { "title": "...", "sections": [...] } }
  },
  "Index": { "tools": { ... } }
}
```

## 데이터 흐름

```
요청 → middleware.ts → URL에서 locale 추출 → next-intl 설정
서버 컴포넌트 → getTranslations() → messages/{locale}.json에서 번역 로드
클라이언트 컴포넌트 → NextIntlClientProvider → useTranslations() → 번역 접근
```

## 주의사항

- **ko.json과 en.json 키 구조 반드시 동일** — 키 누락 시 런타임 에러
- 새 기능 추가 시 **양쪽 파일 모두에 번역 추가** 필수
- `t.raw()`: 객체/배열 형태로 번역 데이터 접근 (FAQ, 가이드 콘텐츠)
- `t()`: 문자열 형태로 번역 접근 (일반 UI 텍스트)
- 가이드 콘텐츠(articles)는 번역 파일에 직접 HTML 없이 텍스트로 작성
