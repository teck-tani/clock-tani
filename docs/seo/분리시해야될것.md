# 프로젝트 분리 시 해야 할 것

> clock-tani를 분리한 이후 시행착오를 거치며 수정해온 내용 정리.
> 새 프로젝트 분리 시 처음부터 적용해서 시행착오를 줄이기 위한 참고 문서.

---

## 분리 이후 수정해온 주요 이슈들

### 🔴 1. hreflang 중복 에러 (가장 큰 문제)

**증상:** Google Search Console에서 hreflang 관련 에러 26개 발생

**원인:**
- next-intl 미들웨어가 HTTP 응답 헤더로 hreflang을 자동 생성
- 메타데이터에서도 alternates.languages로 동시 선언 → 중복
- `x-default` hreflang이 `/ko`와 겹쳐서 크롤러 경고

**해결:**
```ts
// src/navigation.ts
export const { ... } = createLocalizedPathnamesNavigation(routing, {
  alternateLinks: false,  // HTTP 헤더 자동 생성 비활성화
});
```
- `x-default` hreflang 전체 제거
- 메타데이터에서 `alternates.languages`를 명시적으로만 선언

---

### 🔴 2. Sitemap 문제

**증상:** 색인이 예상보다 느림, 다국어 페이지 인식 불완전

**원인 및 해결:**
- 루트 URL(`/`) 자체가 sitemap에 없었음 → 추가
- 각 URL에 다국어 버전 선언 누락 → `alternates.languages` 추가
- 수정날짜가 하드코딩 static 값 → 실제 날짜 반영

**sitemap 기본 구조:**
```ts
{
  url: `https://도메인/ko/페이지`,
  lastModified: '2025-xx-xx',
  changeFrequency: 'monthly',
  priority: 0.8,
  alternates: {
    languages: {
      ko: `https://도메인/ko/페이지`,
      en: `https://도메인/en/페이지`,
    },
  },
}
```

---

### 🟡 3. 메타데이터 길이 초과

**증상:** SEO 도구에서 title/description 길이 경고

**해결:**
- EN `title` 60자 이내로 단축
- EN `description` 160자 이내로 단축
- OG 이미지 누락 페이지 → `opengraph-image.tsx`, `twitter-image.tsx` 생성

---

### 🟡 4. 보안 헤더 없음

**증상:** Lighthouse Best Practices 점수 낮음

**해결 (next.config.ts):**
```ts
headers: [
  { key: 'X-Content-Type-Options', value: 'nosniff' },
  { key: 'X-Frame-Options', value: 'SAMEORIGIN' },
  { key: 'X-XSS-Protection', value: '1; mode=block' },
  { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
  { key: 'Permissions-Policy', value: 'camera=(), microphone=(), geolocation=()' },
  { key: 'Strict-Transport-Security', value: 'max-age=31536000' },
]
```

---

### 🟡 5. 홈페이지가 리다이렉트만 함

**증상:** `/` → `/ko` 즉시 리다이렉트로 홈 URL에 콘텐츠 없음 → SEO 불리

**해결:** 홈페이지에 도구 소개 콘텐츠 직접 추가, Footer에 내부 링크 강화

---

### 🟢 6. 성능 / 접근성

**해결:**
- DNS prefetch 추가 (flagcdn.com, googletagmanager.com)
- 이미지 AVIF/WebP 포맷 활성화, `minimumCacheTTL: 31536000`
- 모든 버튼에 `aria-label` 추가
- `:focus` outline 스타일 추가 (키보드 접근성)
- `prefers-reduced-motion` 미디어 쿼리
